Tag: <header>   <\header>

  • It is called Header Tag
  • It define the header section of a document or a section within a document
  • It typically contains introductory or navigational elements for the document or section it belongs to
  • It is often placed at the top of a webpage or at the beginning of a section. It can contain headings, logos, navigation menus, search bars, and other relevant content
  • It is separate from the main content of the page, acting as a wrapper or container for introductory elements
  • It makes easier for users to navigate and understand the page
  • It is a block-type or block-level element
  • It takes up full width available by default. And the content within this tag is displayed with a line break before and after the paragraph
  • It is a semantic tag
  • Example:
    <header>
      <h1>Website Name<\h1>
      <nav>
        <ul>
          <li><a  href="#...">Home<\a> <\li>
          <li><a  href="#...">About<\a> <\li>
          <li><a  href="#...">Contact<\a> <\li>
        <\ul>
      <\nav>
    </header>

Tag: <nav>   <\nav>

  • It is called Nav Tag
  • It is a semantic tag used to define a section of navigation links within an HTML document. It represents a block of navigation-related content, such as menus, navigation bars, or lists of links.
  • It makes easier for users to navigate through the site
  • It is often placed in header section to link to other pages of the site or to parts within a page. It can be placed in footer section to link related documents (may not be local, use absolute reference). It can also be used in other sections
  • It is a block-type or block-level element
  • It takes up full width available by default. And the content within this tag is displayed with a line break before and after the paragraph
  • Example:
    <header>
      <nav>
        <ul>
          <li><a  href="#...">Home<\a> <\li>   (Relative or Internal Link)
          <li><a  href="#...">About<\a> <\li>   (Relative or Internal Link)
          <li><a  href="#...">Contact<\a> <\li> (Relative or Internal Link)
        <\ul>
      <\nav>
    </header>

Tag: <footer>   <\footer>

  • It is called Footer Tag
  • It is a semantic tag used to define the footer section of an HTML document or a specific section within a document
  • It represents the bottommost section of a webpage or a section containing concluding or auxiliary content. It is often used to include information such as copyright notices, contact details, links to related pages, sitemaps, or other supplemental information
  • It is separate from the main content of the page, acting as a container for content that is not part of the main body
  • It is a block-type or block-level element
  • It takes up full width available by default. And the content within this tag is displayed with a line break before and after the paragraph
  • Example:
    <footer>
      <div class="footer-logo">
        <img src="logo.png" alt="Logo">
      <\div>
      <div class="footer-contact">
        <h3>Contact Informatio<\h3>
        <p>Phone: 125-456-7890<\p
        <p>Email: info@example.com<\p>
        <p>Address: 123 Main Street, City<\p>
      <\div>
      <div class="footer-links">
        <h3>Quick Links<\h3>
        <ul>
          <li> <a href="#...">Products<\a> <\li>
          <li> <a href="#...">Services<\a> <\li>
          <li> <a href="#...">FAQ<\a> <\li>
          <li> <a href="#...">Terms of sevice<\a> <\li>
        <\ul>
      <\div>
    </footer>

Tag: <main>   <\main>

  • It is called Main Tag
  • It is a semantic tag introduced in HTML5 used to define the main content area of an HTML document
  • It represents the primary content of a webpage, excluding header, footer, and other peripheral content
  • It should only appear once per document and should contain content directly related to the central topic or purpose of the webpage
  • It is a block-type or block-level element
  • It takes up full width available by default. And the content within this tag is displayed with a line break before and after the paragraph
  • Example:
    <main>
      <h1>Welcome to our Website</h1>
      <p>Here you will find information about our products and services.</p>
      <section>
        <h2>Featured Products</h2>
        <ul>
          <li>Product 1</li>
          <li>Product 2</li>
          <li>Product 3</li>
        </ul>
      </section>
      <section>
        <h2>Latest News</h2>
        <p>Stay updated with our latest news and announcements.</p>
      </section>
    </main>

Tag: <p>   <\p>

  • It is called Paragraph Tag
  • It is used to define a paragraph of text
  • It is a block-type or block-level element
  • It takes up full width available by default. And the content within this tag is displayed with a line break before and after the paragraph
  • It is a semantic tag. It specially denotes a paragraph of text. It indicates that the enclosed text forms a distinct paragraph within the document
  • Example: <p>This is a paragraph of text</p>

Tag: <div>   <\div>

  • It is called Div Tag
  • It is used to define a generic container or division (group of related content or elements) in HTML
  • It is a block-level element
  • It takes up the full width available by default, but it can be styled with CSS to have a specific width and height
  • It is not a semantic tag, meaning it does not carry any specific meaning or indicate the purpose of the content within it.
  • Example:
    <div>
    <p>This is a paragraph within a div<\p>
    </div>

Tag: <section>   <\section>

  • It is called Section Tag
  • It is used to define a section/container of content within an HTML document (like div)
  • It is often used for larger content divisions, such as chapters, articles, or distinct parts of a webpage.
  • It is a block-level element
  • It takes up the full width available by default, but it can be styled with CSS to have a specific width and height
  • It is a semantic tag, it indicates that the enclosed content forms a distint section or cohesive unit within the document
  • The difference b/w div and section is that div tag is a generic container for grouping and styling content, while the section tag is a semantic element used to define standalone larger sections of content with specific themes or topics.
  • The content within the section is related but does not necessarily represent a standalone piece of content. The content within the section contributes to the overall understanding of the page, but it's not meant to be independently distributed or stand alone. (Unlike article tag)
  • Example:
    <section>
    <h2>Introduction<\h2>
    <p>This section provides an overview of the Topic<\p>
    </section>

Tag: <article>   <\article>

  • It is called Article Tag
  • It is a semantic tag which is used to define a self-contained and complete piece of content that can stand alone (like a blog post, news article, or a forum post) making it easily distinguishable and reusable
  • The content within the article has its own meaning and is not necessarily dependent on the surrounding content. The content has its own unique and meaningful topic or subject
  • It is a block-level element
  • It takes up the full width available by default, but it can be styled with CSS to have a specific width and height
  • Example:
    <article>
      <h1>Introduction to HTML5<\h1>
      <p>HTML5 is the latest version of the Hypertext Markup Language used for structuring and presenting web content.<br>
            It introduces new semantic elements like header, nav, and section that provide improved structure and meaning to web pages.<\p>
      <img src="html5.png" alt="HTML5 Logo">
    </article>

Tag: <aside>   <\aside>

  • It is called Aside Tag
  • It is a semantic tag used to define content that is not directly related to the main content of a webpage. It represents a section of content that is considered separate from the main content but still relevant or supplementary to it.
  • It is typically used for sidebars, pull quotes, advertisements, author biographies, related links, or any content that is not directly part of the main flow of the document.
  • It is typically positioned next to or alongside the main content area, providing additional or supplementary information
  • It is a block-level element
  • It takes up the full width available by default, but it can be styled with CSS to have a specific width and height
  • Example:
    <body>
      <header>
        <!-- Header content here -->
      </header>

      <main>
        <article>
          <!-- Main content here -->
        </article>

        <aside>
          <h2>Related Links</h2>
          <ul>
            <li><a href="#">Link 1</a></li>
            <li><a href="#">Link 2</a></li>
            <li><a href="#">Link 3</a></li>
          </ul>
        </aside>
      </main>

      <footer>
        <!-- Footer content here -->
      </footer>
    </body>

Tag: <details>   <\details>

  • It is called Details Tag
  • It is a semantic tag used to create a disclosure widget or a collapsible section on a webpage
  • It represents a container for additional information that can be revealed or hidden by the user
  • It consists of a "summary", which serves as the visible header or title of the collapsible section, and a content area that is initially hidden
  • It is a block-level tag, means it starts on new line and takes up full width available within its parent container. However, its exact display can be modified using CSS styles if desired
  • Example:
    <details>
      <summary>Click to expand</summary>
      <p>This is some additional content that is initially hidden.</p>
    </details>

    This is some additional content that is initially hidden.

Tag: <hr>

  • It is called HR (Horizontal Rule) Tag
  • It is used to create a horizontal line that visually separates content within a section. It represents transition between topics or sections within the content of a webpage
  • It is a self closing tag
  • It is a block-level tag, and does not has semantics
  • Example:
    <section>
      <h2>About Us</h2>
      <p>We are a company dedicated to providing high-quality products and services.</p>
      <hr>
      <h3>Our Team</h3>
      <p>We have a team of experienced professionals who are passionate about their work.</p>
    </section>

Tag: <address>   <\address>

  • It is called Address Tag
  • It is a semantic tag in HTML used to define contact information or address details for the author or owner of a document
  • It is usually placed within the footer or at the end of an article to provide the author's or publisher's contact information
  • It is a block-level tag
  • It does not provide any special styling or formatting by default. Some browsers display the content within address tag in italic style by defualt, but it is browser specific

Tag: <blockquote>   <\blockquote>

  • It is called Blockquote Tag
  • It is a semantic tag used to indicate a block of quoted content
  • It helps visually distinguish (indent, positioned furhter from the margin) quoted content from the surrounding text
  • It can contain paragraphs, headings, lists, and other types of HTML elements
  • It is often accompanied by a <cite> tag to provide credit to the original source
  • It is a block-level tag
  • Example:
    <blockquote>
      <p>"The only way to do great work is to love what you do."</p>
      <cite> - Steve Jobs</cite>
    </blockquote>

"The only way to do great work is to love what you do."

- Steve Jobs

Tag: <cite>   <\cite>

  • It is called Cite Tag
  • It is a semantic inline element used to mark up the reference or title of a creative work, such as a book, article, song, or movie
  • It is typically used within a block of quoted text to indicate the source or origin of the quote
  • It renders the enclosed text in italics, but it can be customised
  • Example:
    <p> According to a study conducted by <cite>Research Institute</cite>, the findings suggest that... <br>
            In his famous novel, <cite>The Great Gatsby</cite>, F. Scott Fitzgerald explores the themes of... </p>

According to a study conducted by Research Institute, the findings suggest that...
In his famous novel, The Great Gatsby, F. Scott Fitzgerald explores the themes of...

Tag: <abbr   title="">   <\abbr>

  • It is called Abbrevation Tag
  • It is a semantic inline tag used to mark up abbreviations or acronyms in an HTML document
  • It represents a shortened version of a word or phrase, and provide a full explanation or expansion of the abbreviation on hover using the title attribute.
  • Example:
    <p>   The   <abbr title="World Health Organization">WHO</abbr>   is a specialized agency of the United Nations.   </p>

The WHO is a specialized agency of the United Nations.

Tag: <time   datetime="YYYY-MM-DDThh:mm:ssZ"> Human readable date or time <\time>

  • It is called Time Tag
  • When we write data or time on our webpage without using time tag, the time or date is treated as plain text without any semantic meanings
  • Time tag is used to add semantic meaning to specific time-related content on a web page.
  • If we use "Time" tag without "datetime" attribute, it will add semantics (means tell the browse that it is time or date) but there will be no machine readable formate of time or date to be used by accessitive devices or other applications
  • The "datatime" attribute is used with "time" tag. "datetime" attribute is used to provide a machine-readable representation of the date and time in the format "YYYY-MM-DDThh:mm:ssZ"
  • It is semantic inline tag
  • Example:
    <p>   The conference will be held on   <time datetime="2023-07-15T09:00">   July 15, 2023 at 9:00 AM   </time>.   </p>

The conference will be held on July 15, 2023 at 9:00 AM.

Tag: <button>   <\button>

  • It is called Button Tag
  • It is a semantic inline tag used to create a clickable button on a webpage
  • It allows users to interact with the webpage by triggering an action or submitting a form
  • The button tag can contain text, images, or other HTML elements that serve as the button's content
  • It can also include event handlers or JavaScript functions to define the behavior when the button is clicked
  • Example:
    <button>First Button</button>
    <button>   <img src="Images/Pak_Army.png" alt="Pak Army Logo">   </button>

Tag: <meter>   <\meter>

  • It is called Meter Tag
  • It is a semantic inline-level tag used to represent a measurement within a predefined range such as a gauge or progress bar, with a minimum and maximum value
  • The value of the measurement is set using the 'value' attribute, while the minimum and maximum values are defined using the 'min' and 'max' attributes, respectively
  • The text content within the meter tag is used as fallback content that is displayed when the browser does not support the meter element or when assistive technologies are used
  • Example:
    <meter>   </meter>
    <p> Current progress: <meter value="75" min="0" max="100">75%</meter> </p>

Current progress: 75%

Tag: <progress>   <\meter>

  • It is called Progress Tag
  • It is a semantic inline tag used to represent the progress of a task or process
  • The value of the progress is set using the 'value' attribute, while the maximum value is defined using the 'max' attribute
  • No text content is defined within the progress tag
  • Example:
    <progress>   </progress>
    <p>   Uploading file:   <progress value="50" max="100"></progress>   </p>

Tag: <iframe>   <\iframe>

  • It is called iFrame Tag
  • It an inline non-semantic element used in HTML to embed another document or webpage within the current document
  • It creates a rectangular region within the page that displays the content of the specified document or webpage
  • The content within the 'iframe' tag is typically specified using the 'src' attribute, which contains the URL of the document or webpage to be embedded.
  • It provides a way to seamlessly incorporate external content while keeping it isolated from the rest of the page's HTML structure
  • It does not support text content within the opening and closing tags
  • The document or webpage being loaded inside the 'iframe' must be hosted on the same domain (or have the same protocol and port) as the parent document that contains the 'iframe'
  • Example:
    <iframe src="https://www.example.com">  </iframe>
    <iframe src="http://dcs.gcu.edu.pk/DCS/"  width="50%"  height="300px">   </iframe>

Tag: <bdo>   <\bdo>

  • It is called the Bi-Directional Override (bdo) Tag
  • By default, text in HTML is displayed according to the directionality defined by the document's language and the user's language settings
  • The 'bdo' tag is used in HTML to override the default text direction of a specific piece of text
  • It is an inline non-semantic element
  • The 'bdo' tag accepts the 'dir' attribute, which can have one of two values:
    1. 'ltr': Specifies that the text should be displayed left to right
    2. 'rtl': Specifies that the text should be displayed right to left
  • Example:
    <bdo dir="rtl">ABC</bdo>
    <bdo dir="rtl">مرحبا بك</bdo>

Tag: <map>   <\map>

  • It is called Image Map Tag
  • It a non-semantic element used to define an image map
  • An image map allows you to associate clickable areas (hotspots) within an image with specific links or actions
  • The 'map' tag must be used in conjunction with the 'img' tag, where the 'img' tag specifies the source (URL) of the image to be used as the image map
  • The 'map' tag contains one or more 'area' tags, each defining a clickable area on the image
  • The attribute 'name' of 'map' tag specifies a name for the image map, which is used to associate the map with the corresponding 'img' tag using the 'usemap' attribute
  • Example:
    <img src="Images/Flag_of_Pakistan.jpeg"   alt="Clickable Image"   usemap="#myMap">
    <map name="myMap">
        <area shape="rect"  coords="0,0,100,100"  href="Videos/National_Anthem.mp4"  alt="Area 1">
        <area shape="circle"  coords="150,150,50"  href="Images/Pak_Army.png"  alt="Area 2">
        <area shape="poly"  coords="200,50,250,150,150,150"  href="Audios/Naara_e_Takbir.mp3"  alt="Area 3">
    </map>
